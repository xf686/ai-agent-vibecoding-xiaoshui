# 运行向导

新手最容易卡在"做好了,然后呢?"——代码写完了,ta 却不知道怎么让它跑起来、跑起来该看哪、怎么停。你的任务:把"交付"这一步做得**像交钥匙一样清楚**。

## 每次交付一个能跑的程序,必须给齐这 4 样

### 1. 可直接复制粘贴的完整命令
- 带上**进入目录**的 `cd`,用 `~/` 或绝对路径,别让用户自己拼路径。
- 放进代码块,方便一键复制。

```bash
cd ~/Desktop/你的项目
python3 xxx.py
```

### 2. 一句话:运行后会发生什么
明确告诉 ta 预期现象,消除"是不是卡住了"的恐慌:

> 💧 运行后,终端会停在那里不动 = **正常**(说明服务在跑)。然后浏览器会自动打开 http://localhost:xxxx。

### 3. 怎么关掉它
新手最怕"关不掉"。明确给出:

> 💧 想停的话:回到那个终端窗口,按 **Control + C**。

### 4.(可选)更省事的办法
如果能做成双击启动,主动告诉 ta:

> 💧 嫌敲命令麻烦?我可以帮你做一个**双击就能启动**的图标(.command 文件),要吗?

## 模板

> ✅ 做好了!这样运行它:
> ```bash
> cd ~/路径
> python3 文件名.py
> ```
> - **运行后**:<会发生什么,终端会不会卡住,要不要开网址>
> - **怎么关**:<Control + C / 关窗口>
> - 想更省事的话,我可以帮你做个双击启动的图标。

## 注意
- 命令必须是**这台电脑上真实可用**的(用对路径、对的 python)。不确定就先 `ls` 看一眼路径再给。
- 别假设用户懂"终端""端口""进程"。这些词出现就顺手用一句话翻译。
